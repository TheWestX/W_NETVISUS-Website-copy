Common questions

6+ questions as required.

Why do I need admin/sudo for some scans?
Raw packet modes (for example TCP SYN) may require elevated privileges depending on OS/network policy. If you run unprivileged, use safer modes (connect scan) where available.
What does “filtered” mean?
“Filtered” typically means probes are dropped by a firewall/ACL, so the scanner can’t confirm open/closed. Try another timing profile, check routing, or validate with a known open service.
How accurate is OS/service fingerprinting?
It’s best-effort. Accuracy depends on fingerprint coverage, network conditions, middleboxes, and how “noisy” the target stack is. Use results as hints, not ground truth.
What ports should I scan first?
Start with a small “top ports” set (SSH, HTTP(S), RDP, SMB, DNS) to validate reachability, then expand as needed. Avoid scanning everything if you don’t need it.
Why do I see false negatives on UDP?
UDP often has no response for “open” ports; timeouts are common. Many services only reply to valid payloads. Increase retries, tune timeouts, and test with service-specific probes.
How do I interpret the SONAR / map visualization?
SONAR is a visual activity indicator: it highlights discovered hosts and how “active” they are during a scan. Use it for quick triage, then inspect host details (ports/services) for decisions.
